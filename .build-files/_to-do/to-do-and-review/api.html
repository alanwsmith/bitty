API

Functionality is controlled by 10 data-* attributes. Four for elements that define the reactivity. Six for component wrappers that do set up and initializaiton.

The four element attributes are:

  • data-b - convience attribute that sends a batch of signals
  • data-c - calls fuctions without sending a signal
  • data-r - defines signals an element receives
  • data-s - sends individual signals

The top level bitty-1-1 components are responsible for set up and initilization. They use these attributes:

  • data-batch - convience attribute that sends a batch of signals before event listeners are attached
  • data-bridge - identifies the .js file to connect to
  • data-call - calls fuctions without sending a signal before event listeners are attached
  • data-listeners - switches the set of event listeners used by the component
  • data-send - sends individual signals before event listeners are attached
  • data-widget - identifies a class to use from the data-bridge file if the default isn't used

  • data-ignore - tell an element or top level components to ignore specific signals (e.g. to prevent a partent from responding to signals sent from child elements)
  • data-include - tell an element or top level component to only respond to the listed signals (i.e. a way to ignore everything else)
  • data-inert - prevents an element or top level compoent from responding to any signals

Auto-generated UUID

  • data-uuid - an ID that's automatically generated for every element and top level component. It's primary purpose is to aid in error handling, but it's available for use in defining funcionality as well
