The data-* Attributes

The example uses three of bitty's data-* attributes:

  • data-connect: loads the default class from a module that's responsible for providing functionality
  • data-send: sets up the button to send a signal named update when it's clicked
  • data-receive: sets up the div to receive a signal named update when it's sent

The Signal Methods

Signals are mapped to methods from the class by name. So, the update signal connects to the update(event, el) method.

Signal methods receive two arguments:

  1. The event that triggered the signal
  2. An element that has a data-receive attribute with the signal's name

Signal Method Events

The event sent to a signal method comes from an .addEventListener() that bitty initializes on the <bitty-2-0> tag. In the example, it's a standard PointerEvent event that gives us access to the event.offsetX and event.offsetY properties.

The elements are native HTML Element objects. The same ones you get form document.querySelector(). Updating them is done through standard methods and properties like .innerHTML, .appendChild, etc...

Foundation Complete

The data-send, and data-receive attributes are the primary tools at the heart of bitty. They connect native events and elements through standardized method calls. By taking that overhead out of the way, bitty lets you focus on how you want your pages to behave.

bitty has other features for changing event listeners, loading data, interacting with other components, etc... But, that's all optional. Once you connect to a module with data-connect all you have to do is fill in methods for the signals in data-send and data-receive to make your page reactive.
