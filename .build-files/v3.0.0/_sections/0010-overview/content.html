The data-* API

bitty's entire HTML API consists of four data-* attributes. They define the signals that map to methods you write to provide functionality. The three attributes used in the example are:

  • data-connect: loads a class that's responsible for providing functionality
  • data-send: sets an element to send signals with a given name
  • data-receive: sets an element to receive signals with a given name

The other attribute is:

  • data-listeners: provides the ability to change which events bitty listens for

Signal Methods

Signals are mapped to methods by name. For example, the coords signal connects to the coords(event, el) method it get-coords.js. Signal methods receive two arguments:

  1. The event that triggered the signal

  2. An element that has a data-receive attribute with the signal's name

That's The Basics

At it's core, that's all bitty does. Connecting events and elements through methods you write to define the functionality.
