• A `.sender` property is added to each incoming event that's sent as the first argument to signal methods.
  • The property is a reference to the element that fired the signal (i.e. the one with the corresponding `data-send` attribute on it.
  • The `.sender` can be the same as the built-in `.target` from the event (e.g. if a button with a `data-send` attribute on it was used).
  • The `.target` element does not have to have a `data-send` attribute on it. For example, a `button` without a `data-send` attribute can be nested inside a `div` that does. In that case, the `button` remains the `.target` but the `div` becomes `.sender`.
  • Events bubble up the DOM. The `.target` element and any number of its ancestors can have `data-send` attributes on them. Each of which will fire a signal when an trigging event occurs.
