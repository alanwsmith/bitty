    Data attribute naming update for clarity

    • The biggest update is changing data-b, data-c, data-r, and data-s to data-batch, data-call, data-recieve, and data-send, respectivally. I originally use the shorter names to reduce the text length. The increased clarity of the longer names is worth the few extra characters.
    • The other big change is removing the leading _ and $ characters from function names. They were originally put in place to create a naming convention that differentiated between functions that were hit with data-call and data-send. In practice, that wasn't necessary.
