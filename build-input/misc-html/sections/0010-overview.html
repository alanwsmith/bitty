Overview

bitty-js is a web component. It provides reactive functionality. For example, this "Click Me" button displays a numeric timestamp.

[!- autoescape false !] [@ data.misc_html["timestamp-demo.html"].raw @] [!- endautoescape !]

bitty-js uses a signaling approach powered by data-* attributes. For example, this is the timestamp demo's HTML: 


  [!- autoescape false !]
  [@- data.misc_html["timestamp-demo.html"].highlighted -@]
  [!- endautoescape -!]
  • The data-module attribute makes the connection to the JavaScript module responsible for providing functionality.
  • The data-send attribute tells the button to send the update signal when its clicked.
  • The data-receive attribute registers the div as a receiver for the update signal.

Signal Functions

Signals map to functions by name. For example, here's the timestamp-demo.js file: 


  [!- autoescape false !]
  [@- data.example_scripts["timestamp-demo.js"].highlighted -@]
  [!- endautoescape !]

The updateTimestamp function is triggered every time the updateTimestamp signal is sent. It runs once for each element with the data-receive="updateTimestamp" attribute.

Signal functions receive two arguments:

  1. An element with the corresponding data-receive attribute
  2. The event that triggered the signal

The event is ignored in the timestamp demo. The receiving div simply has its innerHTML updated with the timestamp text.

The design is intentonally minimal. The goal being to provide a concise foundation then get out of the way.
