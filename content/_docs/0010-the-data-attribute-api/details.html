[! filter markdown|safe !] __bitty-js__ is a web component. It connects HTML elements and events to JavaScript functions. The connections are made via the __bitty-js API__ which consists of six __data-*__ attributes: - data-listeners - data-module - data-receive - data-send - data-use - data-watch Most of the time, you'll only use three: - data-module - data-send - data-receive ##### data-module __<bitty-js>__ components require the __data-module__ attribute. It sets the path to the JavaScript module responsible for providing functionality to the element. [! endfilter !] 
  [!- filter highlight_html|safe -!]

  

  [!- endfilter -!]
[! filter markdown|safe !] The path can be relative: [! endfilter !] 
  [!- filter highlight_html|safe -!]
data-module="modules/example.js"
  [!- endfilter -!]
[! filter markdown|safe !] absolute: [! endfilter !] 
  [!- filter highlight_html|safe -!]
data-module="/modules/example.js"
  [!- endfilter -!]
[! filter markdown|safe !] or relative starting with a leading dot: [! endfilter !] 
  [!- filter highlight_html|safe -!]
data-module="./modules/example.js"
  [!- endfilter -!]
[! filter markdown|safe !] The only requirement of the JavaScript module is that it export a default class: [!- endfilter -!] 
  [!- filter highlight_javascript|safe -!]
// modules/example.js
export default class {
  // etc...
}
  [!- endfilter -!]
[! filter markdown|safe !] ##### data-send Adding a __data-send__ attribute to an element sets it up to send signals. The attribute can be added directly to a __<bitty-js>__ component: [! endfilter !] 
  [!- filter highlight_html|safe -!]

   

  [! endfilter !]
[! filter markdown|safe !] The value of the __data-send__ attribute is the name of the signal. It maps to a function in the JavaScript module with the same name: [! endfilter !] 
  [!- filter highlight_javascript|safe -!]
export default class {
  componentSignal(_el, _event) {
    // etc...
  }
}
  [!- endfilter -!]
[! filter markdown|safe !] A __<bitty-js>__ component with a __data-send__ attribute sends the named signal when it's initialized. The values the function receives are determined by __data-receive__ attributes (covered below). The __data-send__ attribute can also be applied to child elements of a __<bitty-js>__ component: [! endfilter !] 
  [!- filter highlight_html|safe -!]

  

  [! endfilter !]
[! filter markdown|safe !] The value of the __data-send__ attribute maps to a function with the same name in the JavaScript module. [! endfilter !] 
  [!- filter highlight_javascript|safe -!]
export default class {
  buttonSignal(_el, _event) {
    // etc...
  }
}
  [!- endfilter -!]
[! filter markdown|safe !] Child elements with a __data-send__ attribute send their named signals based on events. The default events are "click" and "input". They can be changed with the __data-listeners__ attribute details below. The values the function receives are determined by __data-receive__ attributes (covered below). [! endfilter !]
