### Version: 7.0.0-rc1

- This update is a set of API refinements to 
make things more obvious and consistent. 


- Moved `ev.sender` to `el.sender` The sender 
element is no longer appended
to the event. It's added to the element instead
so that all general usage can be done through
the element (e.g. `el`) argument.

- Moved `el.getString(KEY)` to `el.stringData(KEY)` 
- Moved `el.getInt(KEY)` to `el.intData(KEY)` 
- Moved `el.getFloat(KEY)` to `el.floatData(KEY)` 

- Added: `el.targetStringData(KEY)` - Removed `ev.target.getString(KEY)` 
- Added: `el.targetIntData(KEY)` - Removed `ev.target.getInt(KEY)`

- Removed `ev.target.getFloat(KEY)` 
- Added: `el.targetFloatData(KEY)`

- Removed `ev.target.stringValue(KEY)` 
- Added: `el.targetStringValue(KEY)`

- Removed `ev.target.intValue(KEY)` 
- Added: `el.targetIntValue(KEY)`

- Removed `ev.target.floatValue(KEY)` 
- Added: `el.targetFloatValue(KEY)`

- Removed `ev.sender.getString(KEY)`
- Added: `el.senderStringData(KEY)`

- Removed `ev.sender.getInt(KEY)` 
- Added: `el.senderIntData(KEY)`

- Removed `ev.sender.getFloat(KEY)`
- Added: `el.senderFloatData(KEY)`

- Removed `ev.sender.stringValue(KEY)` 
- Added: `el.senderStringValue(KEY)`

- Removed `ev.sender.intValue(KEY)` 
- Added: `el.senderIntValue(KEY)`

- Removed `ev.sender.floatValue(KEY)` 
- Added: `el.senderFloatValue(KEY)`
