[! filter markdown|safe !] - **data-module** connects to the file responsible for functionality. - **data-send** tells the **button** to send an **update** signal. - **data-receive** registers the **div** to react to **update** signals. #### Functionality **bitty-js** uses the values from **data-send** and **data-receive** to map to functions with the same name. [! filter markdown|safe !] - The **update** function handles signals from elements with **data-send="update"** - Elements with **data-receive="update"** are processed by the **update** function. The two arguments passed to a **bitty-js function** are: 1. The element to update 2. The event that triggered the update The event is ignored in the timestamp demo. The **div** element has its **innerHTML** updated with the timestamp. #### As Simple As I Can Make It The goal of **bitty-js** is to make reactive pages with as little friction as possible. To wit, **bitty-js** is: - A single file web component - Written in vanilla JavaScript - Powered by standard HTML attributes - Designed for both progressive enhancement and API based content loading - Stylable via the light DOM #### Feedback Requested **bitty-js** is currently **beta version 0.3.0**.
