### Version 8.0.0-beta-1

DATE TBD

This is a big update based off doing more
projects with bitty. Part of it is 
a refinement of the api to simplify
it (there is now only `data-connect`,
`data-listeners`, `data-send`, 
and `data-receive`). The other
part is quality of life improvements
like the `.unwrapOr(VALUE)` for `.getXXX()`
methods that provide the response value
or an alternate value if the request
generated an error.

- TODO: Created `this.api.sleep(MS)` function

- Deprecated and removed `data-use`. It added 
too much mental overhead since it created
a different send/receive mode that only 
affected a single element. That can 
be done with the various `.matchXXX` methods
in a way that's simpler to reason about. 

- TODO: Deprecated `data-init` It was more confusing
to work with than just using `data-send` and
`data-receive` and using `bittyReady()` to run
things for initialization.
