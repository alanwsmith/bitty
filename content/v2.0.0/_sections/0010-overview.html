How It Works

The data-* API

bitty's entire HTML API consists of fourdata-* attributes. They define signals that map to functions to add reactivity to a page. The three used in the example are:

  • data-connect: loads a class that's responsible for providing functionality
  • data-send: sets an element to send signals with a given name
  • data-receive: sets an element to receive signals with a given name

The other attribute is:

  • data-listeners: provides the ability to change which events bitty listens for
Signal Methods

Signals are mapped to methods name. For example, the getCoords signal connects to the getCoords(event, el) method of the get-coords.js module.

Signal methods receive two arguments:

  1. The event that triggered the signal

    They come from a document.addEventListener() for each type (click and input by default). In the example, it's a standard PointerEvent event that comes from a click that provides the event.offsetX and event.offsetY properties.

  2. An element that has a data-receive attribute with the signal's name

    Each element with a data-receive attribute for the signal is sent through the method individually. In the example, it's the <div data-receive="getCoords"> element which becomes the el argument in getCoords(event, el) {}.

That's The Basics

At it's core, all bitty does is connect events to elements through a function. The approach is simple, but powerful. By providing the connection then getting out of the way it lets you focus on the import part: deciding what you want your page to do.
