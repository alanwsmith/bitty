Docs - Signal Method event Arguments

The methods that handle signals take two arguments: 1. the event that triggered the signal, and 2. an element that' receiving the signal. For example:


        [!- filter highlight_javascript|safe !]
signalHandler(event, el) {
  el.innerHTML = "updated";
}
        [!- endfilter !]

This section discusses the events.

  • bitty uses event listeners on the `window` object to catch events from signals.
  • The events passed to the methods are extended from standard DOM Events.
  • The event's `.target` property is the element that triggered the signal.
  • Events from signals are modified to add a `.sender` property. The value is a reference to the element with the `data-send` attribute that sent the event.
  • If the element that triggered the signal has a `data-send` attribute, then it populates both the `.target` and `.sender` properties.
  • If the element that triggered the signal does not have a `data-send` attribute, then it's only referenced in the `.target` property. bitty goes up the DOM searching for the nearest ancestor that has a `data-send` attribute. When it finds one, that element is used to populate the `.sender` property and the event is dispatched.
