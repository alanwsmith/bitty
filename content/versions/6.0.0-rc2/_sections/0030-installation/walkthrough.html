Step-by-Step

  1. Start by adding the bitty component tag.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">
    2. [#

  2. Add a button with a div to generate and hold the output.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  3. Add a data-send attribute to the button with the value showDate to define the signal name.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  4. Add a data-receive attribute to the div with the value showDate to define the signal to listen for.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  5. bitty maps signals to methods inside a class with the same name. The class can be imported from an external file or stored directly in a <script> tag which is what we'll do for this example:

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  6. By default, bitty looks for a class on the window objected named BittyClass.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  7. The showDate signal from the data-send and data-receive attributes maps to a method with the same name.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  8. The two arguments passed to the showDate method are: 1. the event that triggered the signal, and 2. the element with the data-receive attribute that's receiving the signal.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  9. This example ignores the event and only uses the incoming element in the el argument. It's a standard HTML Element with all the properties and methods. In this case, .innerHTML is used to set the contents of the <div> to the current date.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">

  10. The final step is to load the bitty component on the page with a <script> tag.

    <[@ bitty_tag @] data-connect="BittyCodeBlockMaker">
    11. #]
