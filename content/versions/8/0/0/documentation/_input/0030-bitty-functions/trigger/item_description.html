[! filter markdown|safe !] - [] Used to trigger a signal from inside another function. - [] Can trigger one single or multiple. - [] The first argument passed to a triggered signal is a custom event with the type `bittytrigger`. - [] The internal details of the `bittytrigger` event type is an implementation detail with no public API and may change without notice. - [] The second argument argument to a triggered signal that normally holds the sender is `null`. - [] If a signal is triggered and there are corresponding receiver elements (i.e. they have a matching `data-r` attribute), each one is processed individually (the process is the same as what happens with `data-s`). - [] If a signal is triggered and there's no corresponding receiver it is run a single time without an element passed to the third argument (this is the same as `data-s` as well). [! endfilter !]
