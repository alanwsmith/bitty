• Text is not sanitized or checked for validity. Security is your responsibility.
  • The `.renderElement(key)` creates a document fragment, populates the fragments `.innerHTML` with the stored text and returns the first element from the result. Errors from direct text content input with `.addElement(key, content)` happen at that point. (That process is very liberal in what it parses. It has the same tolerance for invalid HTML as the result of the DOM processing.)
