• bitty always listens for `click` and `input` events. If you need something to not respond to those events add a `data-listeners` attribute with only the listeners you want to use for the element.
  • The `data-listeners` attributes only effect sending elements (i.e. elements with `data-send` attributes on them).
  • Adding a `data-listeners` to the `<[@ bitty_tag @]>` tag itself does nothing.
  • The specific set of custom listeners is defined based on elements that exist with a `data-listeners` attribute when the module loads. That is, if a new element is added after a bitty module has initialized with a new listener type in a `data-listener` attribute value it won't be picked up.
