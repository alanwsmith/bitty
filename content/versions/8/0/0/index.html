bitty

a front-end framework for the web

getting started

  • bitty is web component loaded via a <script> tag.

  • it uses a data-connect attribute to connect to a JavaScript module.

  • elements with data-s and data-r attributes send and receive signals.

    [! filter block_highlight("html") !] [! include "v8-bits/timestamp.html" !] [! endfilter !]

  • vanilla functions handle the signals.

    [! filter block_highlight("js") !] [! include "v8-bits/timestamp.js" !] [! endfilter !]

  • for example, those blocks produce this:

    [! include "v8-bits/timestamp.html" !]

native events and elements

  • bitty sends native HTML events and elements to functions.

  • the three arguments passed to the functions are:

    1. the event that triggered the signal.

    2. the element with the data-s attribute that sent the signal.

    3. the element with the data-r attribute receiving the signal.

  • The standard names for these arguments used on this site and in the documentation are ev, sender, and el

    For example:

events and signals

  • bitty listens for events and send signals when they happen.

  • elements with data-s attributes sends signals.

  • elements with data-r attributes receive signals.

  • vanilla JavaScript functions whose names matches the signal provide functionality.

  • each element with a data-r attribute for a given signal is passed through the function individually.

    For example:
