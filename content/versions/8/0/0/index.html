bitty

a front-end framework for the web

download [@ vars.bitty_script_name @]

overview

bitty is a lightweight framework for making web pages. Its key features are:

  • Using native HTML attributes and vanilla JavaScript functions
  • Connecting events and elements through signals
  • Providing functions for common tasks
  • Works equally well with progressive enhancement or loading content dynamically
  • Requires no extra tooling or build processes

getting started

Using bitty is straight-forward. You'll need a copy of bitty, an HTML file, and a JavaScript file.

The HTML

For the HTML file:

  • Pull bitty in with a <script> tag.

  • Add a <[@ vars.bitty_tag @]> tag with a data-connect attribute pointing to your JavaScript file.

  • Use data-s attributes to send signals.

  • Use data-r attributes to receive signals.

For example:

[! filter block_highlight("html") !] [! include "v8-bits/timestamp.html" !] [! endfilter !]

The JavaScript

bitty loads the JavaScript file as a module that:

  • Must export a `b` object that bitty can tie into
  • Exports functions that have the same name as the data-s and data-r attributes in the HTML

For example:

[! filter block_highlight("js") !] [! include "v8-bits/timestamp.js" !] [! endfilter !]

native events and elements

  • Functions receive three arguments:

    1. The event that triggered the signal.

    2. The element with the data-s attribute that sent the signal.

    3. The element with the data-r attribute receiving the signal.
