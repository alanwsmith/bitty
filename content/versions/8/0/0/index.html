bitty

a front-end framework for the web

getting started

  • bitty is a front-end framework that uses signals.

  • it's comes in a component loaded with <script> tag.

  • the component uses a data-connect attribute to connect to a JavaScript module.

  • elements with data-s attributes send signals.

  • elements data-r attributes receive signals.

    [! filter block_highlight("html") !] [! include "v8-bits/timestamp.html" !] [! endfilter !]

  • signals are handled by vanilla functions with the same name.

    [! filter block_highlight("js") !] [! include "v8-bits/timestamp.js" !] [! endfilter !]

native elements and events

  • bitty uses native elements and events.

  • functions receive three arguments when an event triggers a signal.

    1. the event that triggered the signal.

    2. the element with the data-s attribute that sent the signal.

    3. the element with the data-r attribute receiving the signal.

  • The standard names for these arguments used on this site and in the documentation are ev, sender, and el.
