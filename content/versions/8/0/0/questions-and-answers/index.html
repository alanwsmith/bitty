[! filter markdown|safe !] ## What is bitty's goal? To make it easier to make things on the web. ## What's the main idea?
Common things should be easy, uncommon things should be possible.
This is [Lea Verou's](https://lea.verou.me/) quote paraphrasing [Alan Key's](https://en.wikipedia.org/wiki/Alan_Kay) "Simple things should be easy, complex things should be possible." ## What approaches support the goal? 1. Minimize the API. 1. Use native HTML and JavaScript. 1. Don't require dependencies. 1. Don't require build tools. ## Why no JSX? JSX increases the barrier to entry for getting started. (You have to pick up a third thing beyond HTML and JavaScript.) ## What about progressive enhancement? bitty was designed specifically to support progressive enhancement. That's part of the reason it relies on native HTML and JavaScript. You can have 100% of your content in HTML with bitty only providing nice-to-have features.
