Overview

bitty is a web component. It makes pages reactive. For example, click this button a few times to get the coordinates of where you clicked.

  }
}

How It Works

The data-* API

bitty's entire HTML API consists of fourdata-* attributes. They define signals that map to functions to add reactivity to a page. The three used in the example are:

  • data-connect: loads a class that's responsible for providing functionality
  • data-send: sets an element to send signals with a given name
  • data-receive: sets an element to receive signals with a given name

The other attribute is:

  • data-listeners: provides the ability to change which events bitty listens for
Signal Methods

Signals are mapped to methods name. For example, the coords signal connects to the coords(event, el) method of the get-coords.js module.

Signal methods receive two arguments:

  1. The event that triggered the signal

    They come from a document.addEventListener() for each type (click and input by default). In the example, it's a standard PointerEvent event that comes from a click that provides the event.offsetX and event.offsetY properties.

  2. An element that has a data-receive attribute with the signal's name

    Each element with a data-receive attribute for the signal is sent through the method individually. In the example, it's the <div data-receive="coords"> element which becomes the el argument in coords(event, el) {}.

That's The Basics

At it's core, all bitty does is connect events to elements through a function. The approach is simple, but powerful. By providing the connection then getting out of the way it lets you focus on the import part: deciding what you want your page to do.

Installation / Getting Started

Since bitty is a web component, there's no framework to install. Using it is done by downloading the bitty-2.0.0-rc2.full.js file, putting it in your sites direcotry tree, and calling it with a script tag like:

<script src="/releases/bitty-2.0.0-rc2.full.js" type="module"></script>

bitty components work by connection to a class that's responsible for providing functionality to the component. That class can be defined directly on the page. Or, it can be stored in an external file. For example, if we make a a file like:

  randomNum(_event, el) {
    el.innerHTML = Math.random();

Sending Without Receivers

      ".no-receiver-output"

    outputEl.innerHTML = this.#count;

Docs - Singnal Method API

Signals names are mapped to method names in the class that provides funcionality to the component. Check out the demo in the overview for a basic example.

The bitty element is exposed to the class providing functionality via this.api. The bitty component adds the following supplemental methods to the standard element methods and properties.

forward(event, signal)
  • calls another method to trigger its signal. The event which triggered the first signal can be sent to the second.
Parameters

  • event

    The event to forward, can be null.

  • signal

    The name of the signal to forward to (required)

Basic Signal Forwarding

Forwarding an event from one signal to another looks like this:

page.html
<bitty-2-0 data-connect="/v2.0.0/modules/forward/basic.js">
  <button data-send="forwardFirstPart">Pass Event For Coords</button>
  <div data-receive="forwardFirstPart">waiting (first)</div>
  <div data-receive="forwardSecondPart">waiting (second)</div>
</bitty-2-0>
/v2.0.0/modules/forward/basic.js
export default class {
  forwardFirstPart(event, el) {
    el.innerHTML = "event recieved and forwarded";
    this.api.forward(event, "forwardSecondPart");
  }

  forwardSecondPart(event, el) {
    el.innerHTML = `x: ${event.offsetX} - y: ${event.offsetY}`;
  }
}

getHTML(url, subs = [], options = {})

  • Retreives an external file as a string of text

  • Performs find and replace updates from the subs argument

  • Creates a template and sets its .innerHTML to the string

  • Returns the cloned node as a document fragment

  • If there's an error retreiving the external content the return value is undefined

Parameters

  • url

    The URL of the file to get as a string

  • subs

    An array of arrays for substitutions to make

  • options

    Options to pass to the `.fetch()` method

Out of Scope

These items are out of scope for bitty:

  • Triggering signals from events on the bitty element itself. (e.g. click events). The purpose of the element is as a wrapper that provides interactivity to its child elements. Not to be a click target itself.

  • Working with the History API

  • Pretty much anything that can be added as an external function once data-functions is in place.

  • Using a s = new State() type object to send signals when it updates. That could be set up if folks want to do it in there classes. It won't be included in bitty directly. The data-send and data-receive attributes can send signals between components already.

History

Reference notes about the evolution of the project.

  • Any earlier prototype limited signals so they stopped at the component by default with a forwarding mechanism that allowed signals to be seen outside. That added complexity to the overall code and the mental model of working with the components themselves.

The approach is now to send signals all the way to the document root and for each component to have it's listeners attached to the document root. That means that if two components try to use the same signal name for different purposes, they'll collide.

Bitty's goal is to make things easier to build. Not to build complex systems. In that line, if a collision occurs the solution is to rename one of the signals. That's much lower overhead than dealing with limiting and forwarding mechanisms.
