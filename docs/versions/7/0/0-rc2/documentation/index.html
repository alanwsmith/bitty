Availability

  • Child elements: No

Details

</body>
</html>

data-init

Availability

  • <bitty-7-0> tags: Yes
  • Child elements: Yes

Details

  • data-init is designed to do the initial load in of elements on a page.
  • Signals defined by the data-init attribute fire once when the component is initalizing.
  • The signals are processed after bittyInit() is called in the class and before data-send signals from the <bitty-7-0> tag are sent.
  • Child elements with `data-init` attributes that are added to a bitty element after it has been initialized _do not_ fire the `data-init` process.
page.html
<bitty-7-0 
  data-connect="/versions/7/0/0-rc2/modules/data-init/example.on.js">
  <div data-init="loadDataInitExample"></div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-init/example.on.js
const template = `<div>the quick brown fox</div>`;

export default class {
  loadDataInitExample(_event, el) {
    for (let i = 0; i < 5; i += 1) {
      const newEl = this.api.makeElement(template);
      el.appendChild(newEl);
    }
  }
}
Example
data-listeners

Availability

  • <bitty-7-0> tags: Yes
  • Child elements: No

Details

The data-listeners attribute changes the events a component listens for. The default events are `click` and `input`. Changing it to listen to `mouseenter` looks like this:

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-listeners-mouseover.js"
  data-listeners="mouseover">
  <div data-send="listenerMouseOver">
    Move Mouse Over
  </div>
  <div data-receive="listenerMouseOver">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-listeners-mouseover.js
export default class {
  listenerMouseOver(_event, el) {
    el.innerHTML = Math.random();
  }
}
Example
Move Mouse Over

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-listeners-mouseover-out.js"
  data-listeners="mouseover mouseout">
  <div data-send="listenerMouseOverAndOut">
    Move Mouse Over
  </div>
  <div data-receive="listenerMouseOverAndOut">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-listeners-mouseover-out.js

data-receive

Availability

  • <bitty-7-0> tags: No
  • Child elements: Yes

Details

Adding a `data-receive` attribute to an element sets it up to receive signals with the given name.

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-receive/single.on.js">
  <button data-send="receiveSignal">
    Click For Random Number
  </button>
  <div data-receive="receiveSignal">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-receive/single.on.js
export default class {
  receiveSignal(_event, el) {
    el.innerHTML = Math.random();
  }
}

-

page.html

    Click For Random Number
  </button>
  <div data-receive="multipleElementSignal">-</div>
  <div data-receive="multipleElementSignal">-</div>
  <div data-receive="multipleElementSignal">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-receive/multiple-elements.on.js
export default class {
  multipleElementSignal(_event, el) {
    el.innerHTML = Math.random();
  }
}

  singleSignal(_event, el) {

Sending Without Receivers

If an event triggers a `data-send` signal and there are no matching `data-receive` elements, then the corresponding method is fired once with a `null` value for the element. For example:

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-send/no-receivers.on.js">
  <button data-send="noReceivers">
    Increment
  </button>
  <div class="no-receiver-output">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-send/no-receivers.on.js
export default class {
  #count = 0;

  noReceivers(_event, _el) {
    this.#count += 1;
    const outputEl = document.querySelector(
      ".no-receiver-output"
    );
    outputEl.innerHTML = this.#count;
  }
}
Example
-

This is useful when the number of times an update needs to occur differers from the number of receiving elements. For example, incrementing by one works if a button is connected to only one element with a matching `data-receive` attribute:

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-send/one-receiver.on.js">
  <button data-send="incrementWithOneReceiver">
    Increment
  </button>
  <div data-receive="incrementWithOneReceiver">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-send/one-receiver.on.js
export default class {
  #count = 0;
  incrementWithOneReceiver(_event, el) {
    this.#count += 1;
    el.innerHTML = this.#count;
  }
}

async Signal Methods

Signal Methods can be `async`

page.html
<bitty-7-0 
  data-connect="/versions&#x2f;7&#x2f;0&#x2f;0-rc2/modules/data-send/async-send.on.js">

  </button>
  <div data-receive="asyncSendDemo">-</div>
</bitty-7-0>
/versions/7/0/0-rc2/modules/data-send/async-send.on.js
export default class {
   async asyncSendDemo(_event, el) {
    el.innerHTML = "Loading...";
    await new Promise(resolve => setTimeout(resolve, 2000));
    el.innerHTML = "Done";
  }
}

Event Argument Helpers

Methods

ev.param(KEY)
  • Looks for a data-KEY attribute from the ev.target element in its ev.target.datasetproperty that matches KEY and returns the value as a string.
  • If the ev.target element does not have a data-KEY the process looks for ancestors with a data-KEY attribute. The first one that's found gets used.
  • If neither the ev.target element or any of its ancestors have the data-KEYattribute the returned value is undefined.
Arguments
KEY Returns the value from a `data-*` where `*` is the `KEY` (e.g. `data-KEY="value"`)

el.param(KEY)
  • Looks for a data-KEY attribute from the el element in its el.dataset property that matches KEY and returns the value as a string.
  • If the el element does not have a data-KEY the process looks for ancestors with a data-KEY attribute. The first one that's found gets used.
  • If neither the el element or any of its ancestors have the data-KEYattribute the returned value is undefined.
