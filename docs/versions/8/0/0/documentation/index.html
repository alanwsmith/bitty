data-* attributes

Overview

  • The HTML side of the bitty API consists of these four attributes.
  • Attributes can be added to HTML elements to send and receive signals through JavaScript methods.

Items

data-connect
Description
  • `data-connect` is an optional attribute that can be added to bitty-8-0 tags.
  • If a `data-connect` attribute exists bitty will attempt to load the module defined by the attribute's value.
  • If a bitty-8-0 tag doesn't have a `data-connect` attribute it will attempt to load a module class from `window.BittyClass` as a fallback.
Examples
Load a default class from `window.BittyClass` if no `data-connect` attribute exists on a `bitty-8-0` tag.
HTML
<bitty-8-0></bitty-8-0>

<div class="test" data-receive="signal_DA553">bug</div>
JavaScript
<script>
window.BittyClass = class {
  bittyReady() {
    this.api.trigger("signal_DA553");
  }

  signal_DA553(_, el) {
    el.innerHTML = "ok";
  }
};
</script>

  signal_9CE5F(_, el) {
    el.innerHTML = "ok";

// This test uses a custom event instead of
// something like mouseover to facilitate
// automated test output.

    super("bittylocallistenertest", { bubbles: true });
  }

</script>

this.api methods

Overview

Items

addCSS(styles)
Description
  • Adds a string of text as a new adopted stylesheet.
  • Returns the new adopted stylesheet
Examples
Load a block of CSS as an adopted stylesheet
HTML
<bitty-8-0 data-connect="ClassAF8D8"></bitty-8-0>
<div class="test" data-receive="verify_signal_AF8D8">bug</div>
JavaScript
<script>
window.ClassAF8D8 = class {
  #newStyles = `:root { --el-AF8D8: crimson; }`;
  #sheetThatWasAdded;

  bittyReady() {
    this.api.trigger("run_signal_AF8D8 verify_signal_AF8D8");
  }

  run_signal_AF8D8(_, __) {
    this.#sheetThatWasAdded = this.api.addCSS(
      this.#newStyles,
    );
  }

  verify_signal_AF8D8(_, el) {
    const result = this.#sheetThatWasAdded.rules[0].cssText;
    if (result === this.#newStyles) {
      el.innerHTML = "ok";
    }
  }
};
</script>

copyTEXT(selector)
Description
  • Copies the text from the selected element.
  • If the element has a `.value` (e.g. it's a form element), then that value is copied.
  • TODO:If the element doesn't have a `.value` the `.innerHTML` is copied instead.
  • Copying is done asynchronously.

NOTE: Because of the way copying text interacts with the operating system's copy/pasteboard these tests must be verified manually. Use the `Test Manually` buttons to copy text from the elements. Then, paste it into another document to verify the correct value was copied.

Examples
Copy text from a textarea (requires manually testing)
HTML
<bitty-8-0 data-connect="Class96769"></bitty-8-0>
<textarea class="el-96769">text from textarea id-96769</textarea>
<button class="manual-test" data-send="signal_96769">Test Manually</button>
JavaScript
<script>
window.Class96769 = class {
  async signal_96769(_, __) {
    await this.api.copyTEXT(".el-96769");
  }
};
</script>

    this.api.trigger("automatic_test_signal_7A367");
  }

  automatic_test_signal_7A367(_, el) {
    this.api.setCollectionLogFunction("debug", (payload) => {
      return {
        custom_key: "custom_value_7A367",
        payload: payload,
      };
    });
    this.api.debug({ object_example: "test_value_7A367" });
    if (this.api.logs()[0].custom_key === "custom_value_7A367") {
      el.innerHTML = "ok";
    }
  }

  // Provide method to re-enable console logging
  // output and displaying messages when the
  // example button is activated manually.

  signal_7A367(_, __) {
    this.api.setOutputLogLevel("debug");
    this.api.debug("text based log from custome example 7A367");
    this.api.debug(
      { exampleLogObject: "7A367", message: "alfa bravo charle" },
    );
  }

  // NOTE: The default log output function expects
  // the content of the log message to be in a
  // `.payload` property on the object passed to
  // it by the collection function. It will output
  // `undefined` to the console if it's not
  // available. If the signature of the object you
  // create with a custom collector function doesn't
  // contain `.payload` you'll likely want to use
  // `this.api.setOutputLogFunction()` to accommodate
  // your object.
};

bug
  • Awaiting `.trigger()` (i.e. `await this.api.trigger(SIGNALS)`) has no practical effect. The `.trigger()` method creates and fires an event which is processed independently by a separate internal bitty method (and, each bit on the page processes the events independently). No attempt is made to communicate async/await status across method or bit boundaries.
  • Ideally a warning would be provided if you try to call `this.api.trigger(SIGNALS)` with `await`. The current approaches for determining if a method was called with `await` from inside the method itself are both unreliable and computationally expensive. Unfortunately, that makes delivering a warning infeasible.

substitutions

Overview

  • The second argument to `this.api.getXXX(template, subs=[])`, `this.api.makeXXX(template, subs=[])`, and `this.api.XXXFromTemplate(template, subs=[])` is an optional array used for find/replace substitutions.
  • There are several valid formations for the items in the array. This section covers them.
