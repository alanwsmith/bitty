data-* attributes

Overview

  • The HTML side of the bitty API consists of these four attributes.
  • Attributes can be added to HTML elements to send and receive signals through JavaScript methods.

Items

data-connect
Description
  • `data-connect` is an optional attribute that can be added to bitty-8-0 tags.
  • If a `data-connect` attribute exists bitty will attempt to load the module defined by the attribute's value.
  • If a bitty-8-0 tag doesn't have a `data-connect` attribute it will attempt to load a module class from `window.BittyClass` as a fallback.
Examples
Load a default class from `window.BittyClass` if no `data-connect` attribute exists on a `bitty-8-0` tag. - [0010-default-bitty-class-on-window]
HTML
<bitty-8-0></bitty-8-0>

<div class="test" data-receive="signal_DA553">bug</div>
JavaScript
<script>
window.BittyClass = class {
  bittyReady() {
    this.api.trigger("signal_DA553");
  }

  signal_DA553(_, el) {
    el.innerHTML = "ok";
  }
};
</script>

Load a block of CSS as an adopted stylesheet - [0010-add-stylesheet]
HTML
<bitty-8-0 data-connect="Class2E507"></bitty-8-0>

JavaScript
<script>
window.Class2E507 = class {
  #newStyles = `:root { --el-2E507: crimson; }`;
  #sheetThatWasAdded;

  bittyReady() {
    this.api.trigger("run_signal_2E507 verify_signal_2E507");
  }

  run_signal_2E507(_, __) {
    this.#sheetThatWasAdded = this.api.addStylesheet(
      this.#newStyles,
    );
  }

  verify_signal_2E507(_, el) {
    const result = this.#sheetThatWasAdded.rules[0].cssText;
    if (result === this.#newStyles) {
      el.innerHTML = "ok";
    }
  }
};
</script>

copy(selector)
Description
  • Copies the text from the selected element.
  • If the element has a `.value` (e.g. it's a form element), then that value is copied.
  • TODO:If the element doesn't have a `.value` the `.innerHTML` is copied instead.
  • Copying is done asynchronously.

NOTE: Because of the way copying text interacts with the operating system's copy/pasteboard these tests must be verified manually. Use the `Test Manually` buttons to copy text from the elements. Then, paste it into another document to verify the correct value was copied.

    // Disable output to avoid testing noise then
    // fire trigger for automatic test.
    this.api.setOutputLogLevel("none");
    this.api.trigger("automatic_test_signal_7A367");
  }

  automatic_test_signal_7A367(_, el) {
    this.api.setCollectionLogFunction("debug", (payload) => {
      return {
        custom_key: "custom_value_7A367",
        payload: payload,
      };
    });
    this.api.debug({ object_example: "test_value_7A367" });
    if (this.api.logs()[0].custom_key === "custom_value_7A367") {
      el.innerHTML = "ok";
    }
  }

  // Provide method to re-enable console logging
  // output and displaying messages when the
  // example button is activated manually.

  signal_7A367(_, __) {
    this.api.setOutputLogLevel("debug");
    this.api.debug("text based log from custome example 7A367");
    this.api.debug(
      { exampleLogObject: "7A367", message: "alfa bravo charle" },
    );
  }

  // NOTE: The default log output function expects
  // the content of the log message to be in a
  // `.payload` property on the object passed to
  // it by the collection function. It will output
  // `undefined` to the console if it's not
  // available. If the signature of the object you
  // create with a custom collector function doesn't
  // contain `.payload` you'll likely want to use
  // `this.api.setOutputLogFunction()` to accommodate
  // your object.
