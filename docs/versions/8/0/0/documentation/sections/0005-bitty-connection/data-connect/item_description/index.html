• Each <> requires a data-connect attribute.
  • The attribute defines the path to the JavaScript module to load.
  • Modules must contain an export const bitty = {} in order to work with bitty.
  • Multiple <> tags can be placed on a single page.
  • Multiple modules can contain functions with the same name. Each will respond to incoming signals regardless of their source. (namespacing is on the roadmap to provide a seperation of concerns.)
